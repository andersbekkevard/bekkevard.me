# Project Guide

This guide explains how the blog is structured and what you typically change.

## 1) High-level architecture

- Framework: Astro (static site generation)
- Content system: Astro Content Collections
- Styling: Tailwind + custom CSS
- Search: Pagefind (generated during build)
- Deploy target: any static host (`dist` output)

Core folders:

- `src/content/blog`: blog posts in Markdown/MDX (year-based folders supported)
- `src/pages`: routes (index, posts, about, search, RSS, markdown endpoints)
- `src/components`: reusable page/UI pieces
- `src/layouts`: top-level page layouts and metadata wrappers
- `src/utils`: helpers (sorting, OG image generation, etc.)
- `public`: static files (images, fonts, favicon, manifest)

## 2) Content model

Blog posts are validated by `src/content.config.ts`.
Main frontmatter fields:

- `title` (string)
- `description` (string)
- `pubDatetime` (date)
- optional: `modDatetime`, `tags`, `featured`, `draft`, `heroImage`, `ogImage`

Create post files like:

- `src/content/blog/2026/my-post-title.md`

## 3) Rendering flow

- `src/pages/index.astro` loads all posts, shows featured + recent
- `src/pages/posts/index.astro` renders the archive list
- `src/pages/posts/[slug].astro` and year variants render single posts
- `src/pages/rss.xml.ts` generates RSS feed from the collection
- `src/pages/search.astro` renders search UI; index comes from Pagefind

## 4) Site identity and branding

Primary branding source:

- `src/consts.ts` (`SITE`, nav links, social links)

Secondary metadata and social behavior:

- `src/constants.ts` (`SOCIALS` used by icon/social components)
- `src/components/StructuredData.astro` (JSON-LD)
- `src/components/BaseHead.astro` (meta tags)
- `public/site.webmanifest` (PWA metadata)
- `src/pages/index.astro` + `src/pages/about.mdx` (hero/about copy)

## 5) Environment variables used

- `PUBLIC_GOOGLE_SITE_VERIFICATION` (optional search console verification)
- `PUBLIC_TWITTER_HANDLE` (optional, for Twitter card metadata, e.g. `@your_handle`)

## 6) Build and verify

Use these commands:

- `npm install`
- `npm run build`
- `npm run preview` (optional local preview of production build)

`npm run build` runs Astro build and Pagefind indexing.

## 7) Personalization checklist

Update these first:

1. `src/consts.ts`
   - `SITE.website`, `SITE.author`, `SITE.title`, `SITE.desc`, `SITE.profile`
   - `SITE.editPost.url` to your repository edit URL
2. `src/constants.ts`
   - enable/disable social links and set real handles
3. `src/pages/about.mdx`
   - your biography and links
4. `public/avatar.jpg`
   - replace with your preferred profile image

## 8) Notes on markdown endpoints

The project includes markdown endpoint pages:

- `/index.md`
- `/about.md`
- `/posts.md`
- `/archives.md`
- `/posts/<slug>.md`

These are generated by route handlers in `src/pages/*.md.ts` and can be kept or removed based on preference.
